Lesson 14

Grammar - Can na několik způsobů

Možná jste už někdy zavadili o termín modální sloveso. O co jde? Modální sloveso je typ pomocného slovesa, které nám pomáhá tvořit věty vyjadřující schopnost, pravděpodobnost, svolení, povinnost a podmínku.

Modální slovesa jako can, could, may, might, should, must, would a shall se nazývající pomocná a modální z toho důvodu, že pomáhají hlavnímu slovesu k vyjádřit různé odstíny významu, a tak ho módují. Nemění svou formu ve třetí osobě ani nepoužívají infinitivní "to" a je tedy velmi snadné je použít. Tvoří však vlastní záporné tvary a nemají obvyklou časovost. Podívejme se nejdříve na modální slovesa can a could.

💪 Ability or skill = schopnosti či dovednost

Nejpoužívanější význam modálního slovesa "can" je schopnost či dovednost. Vyjadřuje, že někdo má určitou dovednost.

🧩 I can code in multiple programming languages.

🧩 She can troubleshoot complex network issues.

🧩 They can design user-friendly interfaces.

🧩 He can optimize database queries to improve performance.

🧩 With the right tools, we can enhance the security of our systems.

✅ Permission = povolení či oprávnění

Máme povoleno něco udělat, případně žádáme o povolení.

🧩 Can I access the server logs to diagnose the problem?

🧩 Can I update the software to the latest version?

🧩 You can use my laptop if your computer isn't working.

🧩 Can I restart the system after installing the patches?

🧩 Students can use the lab computers for their projects.

💨 Possibility = možnost

Za pomocí can můžeme vyjadřovat i predikce nebo odhady. Předpovídáme, že může něco nastat nebo zvažujeme, co by asi tak mohla být příčina, ale jistě nevíme.

🧩 This glitch can be a result of the recent software update. = ale nevíme to jistě, hádáme

🧩 These vulnerabilities can expose the system to cyberattacks. = je tady ta možnost

🧩 Incorrect configurations can lead to data breaches.

🧩 Too many requests can overload the server.

🧩 A power surge can damage the hardware components.

🙏 Requests = žádosti

Žádáme někoho, jestli by pro nás něco neudělal.

🧩 Can you check the status of the cloud server?

🧩 Can you send me the documentation for the API?

🧩 Can you help me set up the remote access?

🧩 Can you verify if the backup was successful?

🧩 Can you review the code changes before we merge them?

🤝 Help offers = nabídky pomoci

Nabízíme, že pro někoho něco uděláme.

🧩 I can assist you in configuring the firewall settings.

🧩 Don't panic, I can help you with this bug.

🧩 I can share my notes from the last standup if you want.

🧩 I can give you a brief overview of the new features.

🧩 Our team can handle the server migration process if the IT department asks for it.

🤔 Jak je to s tím could?

Jistě si pamatujete, že could je minulý tvar od slovesa can a máte pravdu. Použitím modálního slovesa could skutečně umisťujeme děj do minulosti, kupříkladu jsme v minulosti něčeho byli schopni a teď už (třeba) nejsme.

🧩 When I was at university, I could code for hours without taking a break.

🧩 Before the update, the software could support multiple file formats.

🧩 The team could meet tight deadlines when they had fewer clients.

🧩 She could manage the entire database system single-handedly before the company expanded.

🧩 When we first started, we could run the entire application on a single server without any performance issues.

Could však ve skutečnosti můžeme použít i pro situace v současnosti. Tím, že děj jakoby pomyslně odsuneme do minulosti, stane se náš proslov zdvořilejším, protože druhá strana necítí takový nátlak, a tak se často používá pro velmi zdvořilé žádosti, nabídky a návrhy.

🧩 Could you please check the server's status?

🧩 Could you help me debug this code?

🧩 Could you send me the latest build?

🧩 Could you review this documentation and provide feedback?

🧩 We could collaborate on this project for better results. = nabídka pomoci

🧩 You could try restarting the system to resolve the issue. = návrh

🧩 To enhance user experience, we could incorporate some animations. = návrh

🧩 They could use a different algorithm for faster processing. = návrh

Could může vyjadřovat i predikce s menší pravděpodobností, že se stanou, než predikce tvořené can, a jsou tudíž jsou velmi nejisté.

🧩 The code could be more efficient with some optimizations.

🧩 There could be a bug in the latest version causing the error. = možná tam bug je, ale pravděpodobnost je malá

🧩 Our website traffic could increase significantly during the sale. = obáváme se toho, ale jistě nevíme

🧩 The server could go down if we don't address the memory leak.

🧩 The new software feature could be a game-changer for our users.

❌ Záporné tvary a otázky ❓

Modální slovesa se vymykají obvyklým konceptům gramatických časů a mají vlastní zákonitosti, které však nejsou nikterak těžké. Je jen třeba si uvědomit, že je nejde kombinovat s do/does/did/going to/is/are a podobně.

Záporný tvar slovesa can je cannot, který se opravdu píše dohromady jako jedno slovo a jeho rozdělování je velmi častou chybou. Jeho zkrácenou verzí je can't, která se používá častěji, protože není tak formální. Záporný tvar slovesa could je could not, ale téměř vždy se zkracuje na neformálnější variantu couldn't.

🧩 I can't find the file you sent me.

🧩 We cannot access the website, it's down.

🧩 She can't remember her password.

🧩 We couldn't back up the data yesterday.

🧩 The team couldn't fix the bug before the deadline.

🧩 She couldn't connect to the Wi-Fi in the meeting room.

Otázky se tvoří jednoduše pouhým přehozením modálního slovesa na začátek věty.

🧩 Can you help me set up the printer?

🧩 Can you send me the presentation file?

🧩 Can the server handle more users?

🧩 Could you show me how to share a folder?

🧩 Could I get a new keyboard? This one isn't working.

🧩 Could you check if the email was delivered?

🤔 Jak je to s tím "be able to"?

Nejspíš jste už zavadili o konstrukci be able to, kterou mnoho lidí používá zaměnitelně s can/could. I když nějakou extra velkou chybu neuděláte, když tyto dva způsoby zaměníte, ve skutečnosti má tato konstrukce své vlastní použití a používá se pro schopnost něco udělat při specifické příležitosti, zatímco can/could vyjadřuje naši obecnou schopnost, kterou máme pořád. Protože "be able to" není modální sloveso, řídí se zákonitostmi časů, do kterých ho potřebujeme napasovat.

🧩 I can develop web applications using JavaScript. = máme tuto dovednost pořád, zkrátka něco umíme

🧩 I have some extra time so I should be able to fix that bug immediately. = schopnost opravovat bugy je sice naší obecnou dovedností, ale fakt, že právě teď mám čas, což není obvyklé, je specifická jednorázová situace

🧩 They could resolve technical issues without seeking external support. = v minulosti toho byli opakovaně schopní, nejde tedy o jednorázovou situaci, ale dovednost

🧩 Despite the system crash, I was able to retrieve most of the files. = mimořádná situace, měli jsme štěstí a při trochu jiných podmínkách by se nám to nejspíš nepodařilo

🧩 She was able to configure the new software just in time for the presentation. = jednorázová záležitost

🧩 During the cyberattack, our IT department was able to isolate the threat and protect sensitive data.

🧩 We were able to restore the website before anyone noticed the downtime.

🧩 After several attempts, he was able to recover the lost data.

💎 Be able to pro budoucnost a předpřítomnost

Možná jste se už dovtípili, že je nemožné použít "can" pro skutečné vyjádření budoucnosti, i když kontextově některé věty s "can" mohou na budoucnost svým způsobem odkazovat, třeba něco navrhujeme. Pokud tedy chceme říct, že budeme něčeho schopni v budoucnosti, i když teď zatím nejsme, musíme využít právě "be able to."

🧩 I believe that with further training, our team will be able to implement advanced cybersecurity measures.

🧩 Once the new servers are installed, we'll be able to handle more web traffic.

🧩 With more training, she will be able to manage the network.

🧩 After the data migration is complete, analysts will be able to generate more comprehensive reports.

🧩 When the new coding framework is introduced, programmers will be able to build applications more efficiently.

🧩 I haven't been able to update it yet.

🧩 I have been able to successfully integrate the third-party API into our application.

🧩 Since attending the advanced coding workshop, she has been able to debug the software more efficiently.

🧩 By using cloud-based solutions, we have been able to scale our storage capacity as the company grows.

🧩 Our team has been able to enhance the server's security protocols.

🧩 The IT department has been able to implement multi-factor authentication for all users.

Vocabulary - Slovíčka vyjadřující schopnost

Skill [skɪl]

The ability to do something well, typically as a result of experience or training.

🧩 His skill in coding is evident in the projects he has completed.

🧩 Acquiring new IT skills is essential in the tech industry.

🧩 Her communication skill makes her an excellent team leader.

Talent ['tælənt]

A natural ability to do something well.

🧩 She has a talent for identifying software vulnerabilities.

🧩 The company is looking for new IT talents.

🧩 His talent in user experience design is unmatched.

Capability [ˌkeɪpəˈbɪlɪti]

The power or ability to do something.

🧩 This software has the capability to detect malware in real-time.

🧩 Our IT team has expanded its capability to handle more complex projects.

🧩 The system's storage capability will be upgraded next month.

Proficiency [prəˈfɪʃənsi]

A high degree of skill or expertise in a particular area.

🧩 Her proficiency in multiple programming languages is commendable.

🧩 To ensure cybersecurity, we need someone with proficiency in ethical hacking.

🧩 The certification tests the proficiency of IT professionals in network management.

Expertise [ˌɛkspəˈtiːz]

Special skills or knowledge in a particular field.

🧩 His expertise in cloud computing greatly benefits the company.

🧩 The project's success was due to the combined expertise of the IT team.

🧩 We needed her expertise when facing a challenging software issue.

Potential [pəˈtenʃəl]

The inherent ability or capacity for growth, development or coming into existence.

🧩 The new software has the potential to revolutionize our workflow.

🧩 Our IT team sees great potential in implementing AI solutions.

🧩 There's a potential risk if we don't update our cybersecurity measures.

Listening - How will AI change the world?

In the coming years, artificial intelligence is probably going to change your life, and likely the entire world, but people have a hard time agreeing on exactly how. The following are excerpts from a World Economic Forum interview where renowned computer science professor and AI expert Stuart Russell helps separate the sense from the nonsense.

There’s a big difference between asking a human to do something and giving that as the objective to an AI system. When you ask a human to get you a cup of coffee, you don’t mean this should be their life’s mission, and nothing else in the universe matters. Even if they have to kill everybody else in Starbucks to get you the coffee before it closes— they should do that. No, that’s not what you mean. All the other things that we mutually care about, they should factor into your behavior as well.

The problem with the way we build AI systems now is we give them a fixed objective. The algorithms require us to specify everything in the objective. And if you say: Xan we fix the acidification of the oceans? Yeah, you could have a catalytic reaction that does that extremely efficiently, but it consumes a quarter of the oxygen in the atmosphere, which would apparently cause us to die fairly slowly and unpleasantly over the course of several hours.

So, how do we avoid this problem? You might say, okay, well, just be more careful about specifying the objective — don’t forget the atmospheric oxygen. And then, of course, some side effect of the reaction in the ocean poisons all the fish. Okay, well I meant don’t kill the fish either. And then, well, what about the seaweed? Don’t do anything that’s going to cause all the seaweed to die. And on and on and on.

The reason that we don’t have to do that with humans is that humans often know that they don’t know all the things that we care about. If you ask a human to get you a cup of coffee, and you happen to be in the Hotel George Sand in Paris, where the coffee is 13 euros a cup, it’s entirely reasonable to come back and say: Well, it’s 13 euros, are you sure you want it, or I could go next door and get one? And it’s a perfectly normal thing for a person to do. To ask, I’m going to repaint your house, is it okay if I take off the drainpipes and then put them back?

We don't think of this as a terribly sophisticated capability, but AI systems don’t have it because the way we build them now, they have to know the full objective. If we build systems that know that they don’t know what the objective is, then they start to exhibit these behaviors, like asking permission before getting rid of all the oxygen in the atmosphere. In all these senses, control over the AI system comes from the machine’s uncertainty about what the true objective is. It’s when you build machines that believe with certainty that they have the objective, that’s when you get this sort of psychopathic behavior. And I think we see the same thing in humans.

What happens when general purpose AI hits the real economy? How do things change? Can we adapt? This is a very old point. Amazingly, Aristotle actually has a passage where he says, look, if we had fully automated weaving machines and plectrums that could pluck the lyre and produce music without any humans, then we wouldn’t need any workers. That idea, which I think it was Keynes who called it technological unemployment in 1930, is very obvious to people. They think: Yeah, of course, if the machine does the work, then I'm going to be unemployed. You can think about the warehouses that companies are currently operating for e-commerce, they are half automated.

The way it works is that an old warehouse, where you’ve got tons of stuff piled up all over the place and humans go and rummage around and then bring it back and send it off, there’s a robot who goes and gets the shelving unit that contains the thing that you need, but the human has to pick the object out of the bin or off the shelf, because that’s still too difficult. But, at the same time, would you make a robot that is accurate enough to be able to pick pretty much any object within a very wide variety of objects that you can buy? That would, at a stroke, eliminate 3 or 4 million jobs.

There's an interesting story that E.M. Forster wrote where everyone is entirely machine dependent. The story is really about the fact that if you hand over the management of your civilization to machines, you then lose the incentive to understand it yourself or to teach the next generation how to understand it.

You can see WALL-E actually as a modern version, where everyone is enfeebled and infantilized by the machine, and that hasn’t been possible up to now. We put a lot of our civilization into books, but the books can’t run it for us. And so we always have to teach the next generation. If you work it out, it’s about a trillion person years of teaching and learning and an unbroken chain that goes back tens of thousands of generations. What happens if that chain breaks? I think that’s something we have to understand as AI moves forward.

The actual date of arrival of general purpose AI — you’re not going to be able to pinpoint, it isn’t a single day. It’s also not the case that it’s all or nothing. The impact is going to be increasing. So with every advance in AI, it significantly expands the range of tasks. So in that sense, I think most experts say by the end of the century, we’re very very likely to have general purpose AI. The median is something around 2045. I'm a little more on the conservative side. I think the problem is harder than we think. I like what John McAfee, he was one of the founders of AI, when he was asked this question, he said: "Somewhere between five and 500 years. And we're going to need, I think, several Einsteins to make it happen."

Writing - Já můžu, může to tak být, mohl bys?

📝 Napište 10 vět v jakýchkoli kombinacích významů s použitím can, could a be able to.

🧩 I can code in React, but I can't code in Vue.

🧩 I could code in HTML and CSS when I was at grammar school and now I can also use JavaScript.

🧩 Unfortunately, I wasn't able to make my program work.

🧩 I hope that in future I'll be able to do a bit of back end as well.

🧩 It can become a huge problem if we don't address this bug soon.
