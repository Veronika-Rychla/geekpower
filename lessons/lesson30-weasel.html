Listening - C# in 100 seconds

C#, a statically typed, general-purpose programming language, is famous for being the workhorse of the Windows.NET framework. It was created in the year 2000 by Anders Hejlsberg at Microsoft. It was designed as a modern, C-like, object-oriented language and was originally going to be called Cool. Initially, it was criticized as an imitation of Java, but it has evolved into one of the most popular and well-loved languages to this day.

It can be used to build almost anything, like desktop apps on .NET core, cross-platform mobile apps with Xamarin, web applications with Blazor, and video games with the Unity framework. It became open-source software in 2014 and can be used to build apps outside of the .NET framework. C# code is compiled into an intermediate language, which can then be interpreted by the Common Language Runtime, where it can be executed as native machine code on any operating system without the need to recompile.

It's known primarily as an object-oriented language but also supports functional lambda expressions and has the ability to write a declarative query on any data structure with a feature called LINQ. It's a memory-safe language, thanks to garbage collection, although it is possible to create an unsafe context if you want to allocate your own memory with pointers.

To get started, install the .NET Core SDK, then open the terminal to an empty directory and run .NET new to create a new application. That creates a Program.cs file that imports the System namespace at the top, then provides a class with a main function, which is where your code will start executing. Declare a variable by starting with a type, followed by a name and value. By default, a value cannot be null, but you can make it nullable by adding a question mark to the type.

Now, to organize our code and share it in other files, we will wrap it in a namespace. We can then create a new class inside the namespace that has a constructor when the class is instantiated and a destructor when the class goes out of scope. Classes support inheritance, polymorphism, and everything else you would expect in a modern object-oriented language. We can define a property on the class and make it readable with get or writable with set.

The language doesn't have top-level functions but rather methods that are defined as a member of a class. However, it does support lambda expressions or anonymous functions and other functional programming patterns. When it comes to asynchronous programming, it supports a task-based asynchronous pattern, allowing you to write non-blocking code using the async-await syntax. Now, to compile and execute your code, use the .NET run command.

Statically typed

Describes a programming language in which variable types are explicitly declared and thus determined at compile time.

General-purpose

A type of programming language that is designed to be used for writing software in a wide variety of application domains.

Compile

To transform code written in a programming language into a machine code or lower-level form so that it can be executed by the computer.

Execute

To run a program or a piece of code by a computer to perform its instructions.

Object-oriented

A programming paradigm based on the concept of "objects", which can contain data and code: data in the form of fields, and code, in the form of procedures.

Framework

A platform for developing software applications. It provides a foundation on which software developers can build programs for a specific platform.

Open source

Software for which the original source code is made freely available and may be redistributed and modified.

Intermediate language

A form of input code that a compiler or assembler uses to produce object code, which the computer can execute.

Native machine code

The set of instructions executed directly by a computer's CPU.

Lambda expressions

A function or a subroutine without a name that can be used wherever a delegate is valid.

Garbage collection

Automatic memory management that reclaims memory occupied by objects that are no longer in use by the program.

Namespace

A container that allows developers to bundle all related classes, interfaces, functions and methods under a unique name.

Constructor

A special type of subroutine called to create an object. It prepares the new object for use, often accepting arguments that the constructor uses to set required member variables.

Destructor

A method called when an object is destroyed or deleted to perform cleanup tasks.

Inheritance

A mechanism in which one class acquires the properties and behaviors of another class.

Polymorphism

The ability of different objects to respond, each in its own way, to identical messages or function calls.

Asynchronous programming

A form of parallel programming that allows a unit of work to run separately from the primary application thread and notifies the calling thread of its completion, failure, or progress.

Non-blocking code

Code that does not block or wait for a task to complete before moving on to the next task.

Task-based asynchronous pattern

A design pattern which uses tasks to represent asynchronous operations.

The process of transforming code into a form that can be run (compiled) and then running it (executed).
