Lesson 23

Grammar - Let's mix it!

Mixed conditionals (smíšené podmínky) jsou věty, které kombinují dva různé typy podmínkových vět. Používají se k vyjádření složitých podmínek a výsledků, zejména když se podmínka a výsledek týkají různých časů. To může být zvláště užitečné v kontextech IT a programování, kde hypotetické nebo skutečné scénáře často zahrnují kombinaci minulých akcí a současných nebo budoucích důsledků.

Minulá podmínka a současný výsledek

If + past perfect, would + základní tvar slovesa

Kombinace třetí podmínkové věty (minulá podmínka) s druhou podmínkovou větou (současný výsledek).

🧩 If you had implemented a more robust backup system, our data wouldn't be at risk now.

🧩 If the software had been tested more rigorously, it wouldn't have so many bugs.

🧩 If the team had followed the agile methodology, we would see more progress now.

🧩 If the server had been upgraded last year, it wouldn't be so slow today.

🧩 If we had invested in better cybersecurity, our network wouldn't be vulnerable now.

🧩 If they had encrypted the files, the data breach wouldn't be such a disaster now.

🧩 If the user had read the manual, they wouldn't be struggle with the software.

🧩 If the programmer had documented their code, we wouldn't be confused now.

🧩 If the system had been patched regularly, it wouldn't be exposed to this vulnerability.

Současná podmínka a minulý výsledek

If + past simple, would have + past participle

Kombinace druhé podmínkové větu (současná podmínka) s třetí podmínkovou větou (minulý výsledek).

🧩 If the system were more reliable, we would have prevented the outage last week.

🧩 If this computer were faster, I would have completed the coding task yesterday.

🧩 If our network were more secure, the breach would have been avoided.

🧩 If the software were user-friendly, our clients would have had better experience.

🧩 If the database were more efficient, we would have processed the data faster.

🧩 If the algorithm were more accurate, the predictions would have been correct.

🧩 If the interface were more intuitive, users would have found the feature easily.

🧩 If our website were optimized for mobile, we would have attracted more users.

🧩 If the firewall were stronger, it would have blocked the attack.

V IT se často setkáváme s komplexními situacemi, kde je důležité chápat vztah mezi minulými akcemi a jejich současnými nebo budoucími důsledky. Smíšené podmínkové věty umožňují vyjádřit tyto situace přesněji. Při analýze IT problémů nebo při hledání řešení je důležité umět popsat, co se stalo v minulosti, a jak to ovlivňuje současné situace. Při prezentování projektů, reportování výsledků nebo diskusích o potenciálních řešeních je důležité být schopný jasně a přesně vyjádřit složité myšlenky. Smíšené podmínkové věty jsou pro to ideální.

Vocabulary - Speciální symboly

Equal = /ˈiː.kwəl/

Assignment operator in most languages, equality comparison (as == or ===).

🧩 Assign a value: int x = 10

🧩 Equality test in if statement: if (a == b)

Plus + /plʌs/

Addition in arithmetic operations, string concatenation in many languages, unary plus, increment operator (++).

🧩 Add two numbers: a + b

🧩 Increment a variable: i++;

Dash/Minus - /dæʃ/ or /ˈmaɪnəs/

Subtraction in arithmetic operations, unary negation, decrement operator

🧩 Subtract 4 from 10: 10 - 4

🧩 Negate a value: int x = -5

🧩 Decrement a variable: counter--

Asterisk * /ˈæstərɪsk/

Multiplication in arithmetic operations, pointer declaration in languages like C and C++, wildcard in some contexts.

🧩 Multiply 5 by 3: 5 * 3

🧩 Use as a wildcard in search: *.jpg to find all JPEG files.

Percent % /pəˈsent/

Modulus operator in arithmetic, string formatting in some languages.

🧩Modulus operator: int remainder = 10 % 3

🧩 String formatting in Python: "%s is %d years old" % (name, age)

🧩 CSS for specifying dimensions: width: 50%;

Square brackets [ ] /skweə ˈbrækɪts/

Array indexing, list comprehensions in some languages.

🧩 Access an array element: array[2]

🧩 List comprehension in Python: [x for x in range(5)]

Curly braces or Curly brackets { } /ˈkɜː.li breɪsɪz/

Block definition in many programming languages, object literals in JavaScript, dictionary comprehensions in Python.

🧩 Initialize an object in JavaScript: let obj = {};

🧩 Use in CSS to define styles: body { ... }

Parentheses ( ) /pəˈrenθəsiːz/

Function calls and definitions, precedence grouping in expressions, tuples in Python.

🧩 strings: print("Hello, World!");

🧩 Group expressions: (a + b) * c

Angle Brackets < and > /ˈæŋ.ɡəl ˈbræk.ɪts/

Comparison operators, template syntax in C++, HTML tags.

🧩 In HTML structures

🧩 Comparison operator: 5 > 3, 3 < 5

Ampersand & /ˈæmpəsænd/

Logical AND in some languages (as &&), address-of operator in C/C++.

🧩 Logical AND: if (a && b) { ... }

🧩 Address-of operator: int* ptr = &var

Pipe | /paɪp/

Logical OR in some languages (as ||).

🧩 if (a || b) { ... }

🧩 Pipe in shell commands: ls | more

Exclamation mark ! /ɪkˈskləˈmeɪʃən mɑːk/

Logical NOT, factorial in mathematics.

🧩 Logical NOT: if (!condition) { ... }

🧩 Assert statement in some languages: assert(!isEmpty);

Caret ^ /keərət/

Bitwise XOR, exponentiation in some languages.

🧩 result = 2 ^ 3

Forward slash / /ˈfɔː.wəd slæʃ/

Division in arithmetic operations, path separator in file systems, comment delimiter in languages like C, C++, and Java.

🧩 Divide two numbers: 10 / 2

🧩 In a file path: /home/user/document

🧩 Start a comment in JavaScript: // This is a comment

Backslash \ /ˈbæk.slæʃ/

Escape character in strings, file path separator in Windows.

🧩 Escape character in strings: "He said, \"Hello\""

🧩 File path in Windows: C:\Users\Name

Underscore _ /ˈʌn.dəˌskɔːr/

Variable naming (especially in languages favoring snake_case), private member indicator in some OOP languages, wildcard in some contexts.

🧩 Snake case naming: muj_projekt.js

🧩 BEM in CSS class naming: button__icon--large

Colon : /ˈkəʊ.lən/ and Semicolon ; /ˌsem.iˈkəʊ.lən/

Colon often used in ternary operator, object property definition in JavaScript; semicolon as statement terminator in many languages.

🧩 JavaScript object, JSON? let obj = { name: "John", age: 30 };

🧩 In programming to terminate statements: int x = 5; int y = 10;

Hash # /hæʃ/

Preprocessor directive in C and C++, comment in scripting languages like Python, ID selector in CSS.

🧩 # This is a comment

🧩 #header { color: blue }

Tilde ~ /ˈtɪldə/

Home directory shorthand in Unix systems, approximately equal, pattern matching.

🧩 ~/Documents refers to the Documents folder in the user's home directory.

Listening - How virtual reality tricks your brain

These people are panicking because they’re being asked to step out of an elevator onto a single plank that’s suspended 80 stories in the air. Oh, man. This game is called Richie’s Plank Experience and by their very real reactions, you might expect some groundbreaking, photorealistic visuals behind that headset.

But that’s really not the case. Sure, it’s nice, but it’s clearly not real. The graphics are really beautiful, but the lighting isn’t quite right; things are just a little too diffused. The buildings are almost too smooth. The plants on the ground are clearly squared off. It’s hard to believe these people aren’t dramatizing their reactions for the camera.

But it’s not just the plank experience. The internet is full of VR fails, where people are sucked into pixelated worlds with disembodied parts in front of them. But this really got me thinking. How is it that a virtual world that doesn’t look very real at all, feel so real?

Of course, the first major difference between watching VR and being in it is putting on the headset. Here's where VR marketing has mostly failed. Right? Because it's always shown from a spectator's perspective. And as a spectator, you do not get the experience of the person in VR. Thong Nguyen is the founder and CEO of Roomera, a company that helps businesses test and understand new spaces before they're built using virtual reality.

VR communicates to your brain in a different way than looking at a screen. When looking at a screen — like our TV or phone — our brains read this as a flat image, in the same way we would view a picture. If an object on a screen gets bigger or smaller, or a person on TV moves toward the camera, you don’t feel the need to take a step back or move out of their way, but in VR you might want to, because you’re not looking at one screen. You’re looking at two. And those two screens are literally right in front of your eyes, projecting a slightly different image into each eye.

Which is how our vision works in the real world — each eye takes in stimuli from a slightly different vantage point. You can test this: Hold your finger up in front of your face and wink each of your eyes. Your finger should “jump” back and forth. That “jump” is the difference between what your left and right eye are seeing. The differences between what your eyes see conveys depth. It conveys three dimensions. This is known as stereopsis, and VR developers have spent a lot of time perfecting it.

But what use is depth if you can’t move through it? Which brings us to the next most important way virtual reality tricks your brain: you are the camera, and it's fast enough where your brain starts interpreting it as your perspective. Head tracking allows a person in VR to look and move around a fake world in the same way we look and move around a real one. If you look left, you’ll see more of the world to your left. And if you look down… Okay, maybe don't look down.

Other subtle effects make virtual spaces feel more real: Like 360 audio, which is a big part of the plank experience. As you turn your head, the wind will subtly change. The plank creaks, and if you listen closely, a heart beat slowly begins to speed up. Our brain takes all this new, virtual stimuli and begins to believe that this is reality. Our brain has never really learned within the last one hundred fifty thousand years to actually distinguish between computer generated content and the real world.

Dr. Frank Steinicke has been studying immersive technologies like VR for nearly 20 years. If all the cues that we perceive from the virtual environment are so similar to the cues that we get in the real world, it makes sense that we are unable to clearly distinguish between both. Our brains quickly adapt to virtual environments largely because it’s wired to trust our sense of sight. There's some research showing that approximately 80 percent of all the information that we perceive from our environment are based on vision only.

And this allows VR developers to manipulate our reality even further: what we found out about 10 years ago is that if we guide users on a circular arc with a radius of 20 meters, they have no chance to identify that. They actually walk in a circle in the real world when they see a straight path in the virtual environment. You can walk an entire virtual city without ever leaving a room.

Once we believe the environment is real and accept that we’re actually in it, our brains then go on to fill in some other blanks: there is interesting findings that if you are in the virtual world in a very, let's say, snow or icy environment, people feel cool, although they're in the real world and maybe in a hot environment.

Outside of games, VR has shown a lot of promise in the medical world from reducing pain for burn victims by immersing them in a snowy world while their bandages are changed and with exposure therapy that helps people with phobias — like a fear of heights — and body dysmorphia. It’s also been used for physical therapy like assisting elderly people with their balance.

Right now, we’re still tethered to a system with a headset on our faces. The graphics are nice, but not perfect — and so we’re not fully immersed when we hover 80 stories above the ground. That is...we can still remember to take the headset off. Steinicke cautions that it might not always be that way. We can easily assume that within the next five to 10 years or so, we will not be able to distinguish visually computer generated content from real world content anymore. And then, of course, there are a lot of ethical questions.

For now, VR might not look exactly like reality, but it follows a lot of the rules our brain has learned to perceive as real, and that’s often enough to make us sweat.
