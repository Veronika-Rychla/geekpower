Lesson 19

Grammar - Podmínky reálné aneb příčina a následek

Podmínky jsou if -> then (pokud něco -> pak něco) konstrukce, které typicky zahrnují nějakou podmínku a výsledek vycházející z toto podmínky, pokud (ne)bude naplněna. Porozumění podmínkovým větám může značně zlepšit naši vyjadřující schopnost a to hlavně v technickém kontextu, kde na přesných instrukcích opravdu záleží.

🎯 Proč potřebujeme umět podmínkové věty?

Zejména technická dokumentace často obsahuje podmínky, zejména pokud jde o troubleshooting nebo uživatelské manuály. Například když potřebujeme vyjádřit situace jako "Pokud natrefíte na error X, proveďte akci Y. V takovýchto případech si musíme být jistí, že je instrukce jasná a bude správně pochopena. K tomu nám nejlépe poslouží právě podmínkové věty.

Pochopení podmínek je také nutné k efektivnímu řešení problémů a vymýšlení alternativním scénářů. Kolikrát jste určitě uvažovali stylem "když změním tuhle část kódu, co se stane? If I change this part of the code, what will happen?

Uvažování v podmínkových větách je důležité také pro projektový management, kdy zvažujeme různé scénáře, jak se může náš projekt vyvíjet. "If we don't meet the deadline, what will be the impact?" nebo "If we implement this feature, how will it affect the system stability?" nebo "If we allocate more resources to this task, we could finish earlier." 🤔

Samotné programování je často chápáno jako série podmiňovacích výrazů. Kupříkladu porozumění, jak logické výrazy typu "if-then-else" fungují je téměř totožné lingvistikému chápání této problematiky. Tato paralela nám pak může pomoci jak při programování, tak při používání angličtiny.

🎆 Zero conditional

If + present simple, present simple

Zero conditional se používá k vyjádření obecných pravd, vědeckých faktů nebo situací, které jsou vždy pravdivé za určitých podmínek. Jedná se o způsob, jak vyjádřit, že kdykoli je splněna určitá podmínka, vždy následuje konkrétní výsledek. Tato forma se často používá v situacích, kdy výsledek je vždy stejný a nemění se.

🔬 Obecné pravdy a vědecká fakta, které jsou všeobecně přijímány jako pravdivé

🧩 If you heat ice, it melts.

🧩 If you mix red and blue, you get purple.

🧩 If you click 'Save', the document stores the current changes.

🧩 If electricity flows through a conductor, it generates heat.

💨 Zvykové nebo typické reakce na konkrétní situace

🧩 If I drink coffee late at night, I don’t sleep well.

🧩 If I exercise in the morning, I feel more energetic throughout the day.

🧩 If people smile at me, I usually smile back.

📐 Instrukce a pravidla, která vždy platí

🧩 If the light is green, you can go.

🧩 If you see a 'Wet Floor' sign, walk carefully to avoid slipping.

🧩 If the alarm rings, exit the building immediately.

🧩 If the temperature outside is below freezing, wear a coat.

🧩 If you borrow a book from the library, return it within two weeks.

🧩 If you cross the street, look both ways for traffic first.

Ve všech těchto případech Zero Conditionál vyjadřuje jistotu – pokud je podmínka splněna, výsledek je vždy stejný. Jedná se o přímý a faktický způsob mluvení, který se často používá v technických oborech k popisu konzistentních výsledků na základě určitých podmínek, a tak není divu, že se v touto podmínkou v IT setkáte skoro na každém kroku

🧩 If you press Ctrl + C, it copies the selected text.

🧩 If a user inputs invalid data, the program displays an error message.

🧩 If a computer overheats, it shuts down automatically.

🧩 If code is not properly commented, it becomes hard to understand.

🧩 If an algorithm is efficient, it processes data faster.

🎇 First Conditional

If + present simple, will + base verb

First Conditional se používá k vyjádření skutečných nebo možných situacích, které by mohly nastat v budoucnosti. Je to způsob, jak vyjádřit události, které nejsou zaručené, ale jsou považovány za možné nebo vysoce pravděpodobné za určitých podmínek.

💫 Predikce o budoucnosti na základě současných nebo předvídatelných podmínek

🧩 If it rains tomorrow, we will cancel the picnic. = Podmínka je "it rains tomorrow" a výsledek je "we will cancel the picnic."

🧩 If the server overloads tonight, we will have to reboot it.

🧩 If the traffic to our website increases, we will need more bandwidth.

🧩 If the new update releases tomorrow, we will begin the installation process.

🧩 If the cloud storage reaches its limit, we will need to purchase additional space.

🧩 If the client approves the design, we will start the development phase.

🧩 If the database query runs slowly, we will optimize the code.

🧩 If the system passes all tests, we will deploy it next week.

🧩 If users report bugs after the release, we will issue patches immediately.

📅 Plány a rozhodnutí, které závisí na současné situaci

🧩 If I finish work early, I will meet you for dinner. = To naznačuje plán setkat se na večeři, podmíněný dokončením práce brzy.

🧩 If I finish this coding task early, I will start working on the documentation.

🧩 If we complete the project ahead of schedule, we will plan a team celebration.

🧩 If the test results are positive, I will schedule a demo for the stakeholders.

🧩 If I learn this new programming language, I will apply for the software developer position.

🧩 If the team agrees on the proposal, we will implement the changes next quarter.

🧩 If I master these IT skills, I will lead the next big project.

🧩 If the app receives positive feedback, we will introduce more features.

🚫 Varování před následky možné budoucí události

🧩 If you don’t study for the test, you will not pass. = Varování zde je o následku (neudělání testu) pokud nastane podmínka (nestudování).

🧩 If our cybersecurity measures fail, we will face a potential data breach.

🧩 If you don’t update your antivirus software regularly, you will risk malware infections.

🧩 If we don’t backup our data regularly, we might lose critical information.

🧩 If the team ignores these error logs, we will face serious system issues.

🧩 If users ignore the security protocols, they will compromise their personal data.

🧩 If we don’t test the application thoroughly, we will encounter bugs post-launch.

🧩 If the network is not secured, we will be vulnerable to cyber attacks.

🧩 If the database is not properly maintained, we will experience performance issues.

🙏 Nabídky, návrhy a žádosti na základě určitých podmínek

🧩 If you need help, I will give you a hand. = Nabídka pomoci je podmíněna tím, že osoba pomoc potřebuje.

🧩 If you see John, will you tell him to call me? = Žádost o předání zprávy je podmíněna tím, že posluchač uvidí Johna.

🧩 If the server needs maintenance, I will come in early to work on it.

🧩 If you’re not familiar with the software, the company will provide training.

🧩 If the team needs additional resources, I will talk to the department head.

🧩 If you find the instructions unclear, I will clarify them for you.

🧩 If you need a beta tester, I will sign up.

🧩 If you encounter any error messages, will you document them for the IT team?

🧩 If you finish the analysis early, will you send me the report?

🧩 If you see any system anomalies, will you alert the network administrator?

🧩 If you test the application, will you provide feedback on the user interface?

🧩 If you talk to the client, will you ask for their feedback on our software?

Ve všech těchto případech First Conditional vyjadřuje skutečnou nebo velmi pravděpodobnou situaci v budoucnosti, podmíněnou konkrétní podmínkou v současnosti. Tento kondicionál je praktický pro každodenní použití, zejména při plánování, předvídání nebo přemýšlení o budoucích událostech na základě současných situací nebo pravděpodobných výskytů.

Vocabulary - Slovíčka příčin a následků

Consequence (důsledek) /ˈkɒn.sɪ.kwəns/

Výsledek nebo efekt akce nebo podmínky.

🧩 Neglecting system backups can have severe consequences.

🧩 A positive consequence of automating the process was a significant increase in efficiency.

🧩 One consequence of data breaches is the loss of customer trust.

Condition (podmínka) /kənˈdɪʃ.ən/

Požadavek nebo stav, který musí existovat nebo být způsoben.

🧩 The script runs under the condition that the system passes all preliminary checks.

🧩 A key condition for the software's success is its user-friendliness.

🧩 The warranty is valid on the condition that you don't modify the hardware.

Effect (efekt) /ɪˈfekt/

Změna, která je výsledkem nebo důsledkem akce nebo jiné příčiny.

🧩 The new algorithm had an immediate effect on the processing speed.

🧩 The widespread use of smartphones has had a huge effect on web design.

🧩 The effect of cloud computing on business efficiency has been significant.

Result (výsledek) /rɪˈzʌlt/

Něco, co se stane nebo existuje kvůli něčemu jinému.

🧩 The result of the coding error was a series of system crashes.

🧩 As a result of the team's hard work, the project was completed ahead of schedule.

🧩 The result of insufficient testing was a buggy initial release.

Cause (příčina) /kɔːz/

Osoba nebo věc, která vyvolává akci, jev nebo podmínku.

🧩 The primary cause of the network issue was outdated firmware.

🧩 User error is often the cause of many software problems.

🧩 The cause of the hadrware failure was overheating.

Outcome (výsledek) /ˈaʊt.kʌm/

Způsob, jakým věc dopadne; důsledek.

🧩 The outcome of the system upgrade was a more secure IT environment.

🧩 The outcome of the data analysis will guide our marketing strategy.

🧩 The outcome of the training program included increased employee proficiency in the new software.

Impact (dopad) /ˈɪm.pækt/

Akce jednoho objektu, který silně narazí na druhý; výrazný efekt nebo vliv.

🧩 The impact of artificial intelligence on the industry is tremendous.

🧩 The new regulations had a significant impact on our data handling procedures.

🧩 The introduction of 5G technology will have a huge impact on mobile communications.

Scenario (scénář) /sɪˈnɑː.ri.əʊ/

Písemný nástin děje a jednotlivých scén.

🧩 In a scenario where the system fails, we have a backup plan in place.

🧩 Testing various scenarios ensures that ur software is robust and reliable.

🧩 In the worst-case scenario, we may need to revert to the previous software version.

Sequence (sekvence) /ˈsiː.kwəns/

Konkrétní pořadí, ve kterém související události, pohyby nebo věci následují jeden za druhým.

🧩 The right sequence of commands is critical for the script to run correctly.

🧩 We need to determine the sequence of events that led to the data breach.

🧩 Understanding the sequence of user interactions helps in UI/UX design."

Implication (náznak důsledku) /ˌɪm.plɪˈkeɪ.ʃən/

Závěr, který lze vyvodit z něčeho, i když není výslovně uveden.

🧩 The new privacy policy has far-reaching implications for user data security.

🧩 One implication of rapid tech advancement is the constant need for skill updating.

🧩 The widespread adoption of artificial intelligence has profound implications for job markets, as it may lead to the automation of many current positions.

Cause (způsobit) /kɔːz/

Způsobit, aby se něco stalo, obvykle něco špatného.

🧩 Incorrect data input can cause errors.

🧩 Overheating can cause serious damage to the computer's processor.

🧩 A minor glitch in the code caused a major malfunction in the application.

Result in (mít za následek) /rɪˈzʌlt/

Stát se nebo existovat jako výsledek něčeho.

🧩 The system upgrade resulted in better performance.

🧩 The successful marketing campaign resulted in increased sales.

🧩 The team's effort resulted in the project's early completion.

Affect (ovlivnit) /əˈfekt/

Mít vliv/efekt na něco.

🧩 Emerging technologies significantly affect how we approach problem-solving.

🧩 Budget cuts will likely affect the scope of our IT projects.

🧩 The network outage affected all the office computers.

Influence (ovlivnit) /ˈɪn.flu.əns/

Mít vliv na charakter, vývoj nebo chování někoho nebo něčeho.

🧩 Advancements in computing power have influenced software development trends.

🧩 User feedback can greatly influence the design of an application.

🧩 Market trends influence our company's approach to new technology.

Lead to (vést k) /liːd tuː/

Vést k něčemu.

🧩 Effective cybersecurity measures lead to reduced risk of data breaches.

🧩 Regular updates lead to better software performance.

🧩 Innovations in technology lead to new opportunities in various fields.

Contribute (přispět) /kənˈtrɪb.juːt/

Pomáhat způsobit událost nebo situaci.

🧩 Effective communication contributes to the success of remote teams.

🧩 A well-organized database contributes to efficient data retrieval.

🧩 Employee feedback contributes to the continuous improvement of IT policies.

Trigger (spustit) /ˈtrɪɡ.ər/

Způsobit, aby něco začalo.

🧩 A specific keyword in the email triggered the spam filter.

🧩 The system error triggered an immediate investigation by the IT team.

🧩 Detecting a security threat triggered the firewall's protective measures.

Generate (generovat) /ˈdʒen.ə.reɪt/

Způsobit vznik něčeho.

🧩 The new software generates detailed performance reports.

🧩 This code generates a random sequence of numbers for security purposes.

🧩 Automated scripts generate responses to common user queries.

Provoke (provokovat) /prəˈvəʊk/

Vyvolat reakci nebo emoci, typicky silnou nebo nevítanou.

🧩 The unexpected shutdown provoked a thorough system check.

🧩 The announcement provoked a discussion among the tech team.

🧩 "A breach in protocol provoked a review of security measures.

Imply (naznačovat) /ɪmˈplaɪ/

Silně naznačovat pravdu nebo existenci něčeho, co není výslovně uvedeno.

🧩 The report implies that we need to upgrade our hardware.

🧩 The error message implies a problem with the user's login credentials.

🧩 The data trends imply a growing interest in cloud-based solutions.

Listening - What are conditions?

Let's go think about a day at school or at home. You probably have some rules to follow, right? When you're at school, you likely have a set of classroom rules like where you can put your belongings, how to line up to go to lunch, or when you can leave the classroom. At home, maybe it's a rule that you have to take your shoes off when you come inside, or you're only allowed to use electronics at a certain time.

Computers follow rules too. A computer program runs based on rules written in code by a programmer. It will follow the rules or instructions in the program exactly as they are written. A conditional statement is an "if-then" statement that allows the computer program to change its path and do something different. If something is true, then something else will happen. Using a condition in your code tells the computer to make an exception to a rule.

So, a good example of this is a password. As a rule, a computer will not let you log in without the correct password. It's designed this way for security. But an exception to this rule can be made: if you enter the correct password, then you can enter. Conditions like this are all around you.

Let's say a rule in your house is that on a school day, you can only use electronics for a certain amount of time. But what if your school building is closed and you have to do class online? Then an exception to this rule must be made. If school is online, then you can spend more time on electronics.

In Kodable, we use colors to represent conditions, they can be all different colors. We know our fuzz is going to roll in the direction we tell it to. So if we say "go right," then it will move right until it hits a wall. If we want our fuzz to turn, we can say "if purple, then turn down." The arrow command goes on top of the condition tile, so "if purple, then turn down." After that, we can complete the maze.

Notice how the fuzz changes directions. Now it's your turn. Practice using conditional statements to change your fuzz's path in your program. Watch how the fuzz changes direction when you tell it to with your colored conditioned tiles.
