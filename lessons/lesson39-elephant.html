Lesson 39

Grammar - Časové předložky

Výběr předložek je do značné míry založen na tradičních vzorcích používání, které se časem standardizovaly. Anglické předložky mohou být obzvláště záludné, protože se často spoléhají více na konvenci než na přísná pravidla, a tyto konvence se mohou značně lišit mezi dialekty. 🤯 Dnes se podíváme na časové předložky.

Časové předložky nám pomáhají vyjádřit, kdy se něco děje nebo jak dlouho trvá určitá událost. Předložky jako AT / IN / ON nám umožňují vyjádřit přesný čas, den, datum nebo dobu trvání určité akce a jejich porozumění je klíčové pro správné vyjádření časových vztahů v angličtině.

📆 ON pro dny a data

📑 Dny v týdnu: on Monday / on Friday / on Saturdays

📑 Přesná data: on 22 March 2024

💡 Pokud něco děláme pravidelně určitý den, je možné (a často dokonce stylově lepší) použít tento den v množném čísle. Předložka zůstává stejná.

🧩 I'm usually super busy on Mondays. - I česky říkáme "v pondělky," abychom dali důraz na fakt, že jde obecně o všechny pondělky.

🎇 Co speciální dny v roce a svátky?

Používání předložek jako "on" a "at" s výjimečnými dny v roce v angličtině často závisí na tom, zda je den považován za konkrétní bod v čase nebo obecnější období. Například pro "Christmas Day" typicky používáme "on," protože se jedná o konkrétní den a tedy jedno konkrétní datum v kalendáři. Když ale mluvíme o širším období svátků, zahrnujícím události nebo oslavy předcházející den Vánoc a i následující sváteční dny, používáme "at." Celkově volba mezi "on" a "at" závisí na tom, zda mluvíme o konkrétním dni nebo delším období.

🎄 on Christmas Day ❗ at Christmas

🎁 on my birthday

🐰 on Easter Sunday ❗ at Easter

🎃 on Halloween

💝 on Valentine's Day

🍖 on Thanksgiving Day

🧨 on New Year's Eve

🧩 At Easter, we usually spend time with our family. = celkové období Velikonoc

🧩 We have a big dinner with relatives on Easter Sunday. = konkrétní den

⏳ IN pro delší časové úseky

Předložka IN se používá pro vyjádření delších časových úseků, období nebo časových rámců.

📑 Měsíce: in January / in June / in December

📑 Roky: in 1988 / in 2020

📑 Věky: in the Middle Ages / in the 21st century

📑 Seasons: in spring / in summer / in autumn / in winter

💡 Pokud k dekádě dáme příponu -s a určitý člen "the," označíme tak celé jedno desetiletí.

📑 Dekády: in the 1980s (v osmdesátých letech 20. století)

💡 Nejspíš jste si všimli (a možná vás to zmátlo), že existují dvě slovíčka pro podzim: autumn a fall. Autumn se používá v britské angličtině a je latinského původu. Oproti tomu fall je americká varianta a jde doslova o "fall of leaves" = opadávání listů. 🍃

💫 Trochu nepořádek v různých částech dne

Používání předložek jako "in" a "at" pro různé části dne je ovlivněno jak tradicí, tak způsobem, jakým se angličtina vyvinula k vyjadřování nuancí času. In the morning / in the afternoon / in the evenin používají IN, protože odkazují na časové období, které trvá několik hodin a během toho se něco děje.

"At night" používá "at," protože noc je obecně vnímána ne jako období, ve kterém se něco děje, ale stav dne. "At noon" a "at midnight" používají "at," protože se odkazují na velmi specifické, přesné okamžiky spíše než na delší období. Poledne je přesně uprostřed dne a půlnoc je přesně uprostřed noci.

📑 in the morning

📑 in the afternoon

📑 in the evening

📑 at lunchtime / at noon

📑 at night

📑 at midnight

📑 at sunrise / at sunset

💡 Pokud spojíme dvě určení času, která mají každá jinou předložku, pak vítězí předložka pojící se k prvnímu slovu v pořadí.

📑 on Monday morning

📑 on Friday night

🤔 Jak je to s těmi víkendy?

Pokud jste zmatení, jestli se říká ON nebo AT, protože jste nesčetněkrát viděli obojí, pak vězte, že obojí je gramaticky správně. Záleží však, jakou variantu angličtiny používáte. "At the weekend" se používá v britské angličtině, "on the weeekend" zase v americké.

Co se týče členu, tak pokud použijete určitý člen "the" a weekend necháte v jednotném čísle, zdůrazňuje konkrétní nadcházející víkend. "On weekends / at weekends" bez členu a v množném čísle zobecňuje výrok pro všechny víkendy nebo většinu víkendů.

🧩 What are you doing at the weekend? = Co děláš TENTO víkend? + present continuous pro budoucnost

🧩 What do you usually do at weekends? = Co OBVYKLE děláš o víkendech? + present simple pro opakovanou akci

🌌 Minulost, současnost a budoucnost

📑 in the past / in (the) future

"In the past" používá určitý vždy člen, protože odkazuje na známou minulost, která se již odehrála. Naproti tomu "in future" bez členu má spíše příchuť "from now on," když mluvíme o budoucnosti obecně a nekonkrétně. Nevíme, kdy to bude a co se přesně stane, protože se to stane "někdy neurčitě" v budoucnosti. "In the future" s určitým členem naznačuje, že budoucnost už máme blížeji definovanou a něco konkrétně očekáváme.

🧩 In future, please make sure to submit your reports on time.

🧩 In the future, I plan to continue my education and earn a master's degree.

📑 at present / in the present

Co se týče událostí současných, dává se ve většině případů přednost ustálenému spojení "at present," který má význam "currently / now" a odkazuje na události, které se zrovna dějí. Současnost se sice dá vyjádřit i jako "in the present," ale jde spíše o abstraktní koncept historické éry, a tak tento typ zápisu najdete jen v odborných článcích pojednávající o současné éře.

🧩 At present, I'm too busy to take on any new projects.

🧩 In the present era, digital technology plays a crucial role in shaping global economies, contrasting sharply with the industrial focus of the past.

🌌 Last / every / this / next

Tato časová určení nepoužívají předložky ani nepotřebují členy.

📑 last week / last month / last year / last February

📑 every week / every month / every year / every February

📑 this week / this month / this year / this February

📑 next week / next month / next year / next February

🧩 Last year, our team successfully migrated all our data to the cloud.

🧩 Our company hosts an annual tech summit every year to showcase the latest innovations.

🧩 Our IT department will be attending a conference this February to learn about new trends.

🧩 Next week, we are scheduled to launch the beta version of our new application.

💡 Počkat, in THE next few weeks?

Někdy se můžeme setkat s výrazy jako třeba "in the next few years", které obsahují určitý člen. Jak je to možné? Trik je v tom, jak angličtina pracuje s časovou specificitou.

"Next week" refers to the week after the current one, and "last week" to the week before. Since these terms are generally considered to be singular and specific enough on their own, they do not require an article. They are treated as fixed, known periods which are clearly defined by the words "next" and "last."

"In the next few years" refers to a more extended and somewhat indefinite period. English often uses the definite article "the" to bring more specificity to phrases that are inherently vague. Thus, "in the next few years" specifies a particular span of time within the broader, indefinite future. The article "the" helps to focus on this specific segment of time.

🧩 We will see improvements in the upcoming months. = Refers to the weeks that are approaching, with "the" specifying this near-future period.

🧩 The company has grown significantly in the previous years. = Refers to years in the past, with "the" giving specificity.

📅 Za nějakou dobu

Pokud se něco stane za nějakou určitou dobu, používáme předložku IN.

🧩 The system update will be complete in a few minutes so please be patient.

🧩 In a few years, we expect our AI technology to be fully integrated into our customer support system.

🧩 Our team will launch the website in eight months, after thorough testing.

🧩 We’ll be conducting a network security audit in a couple of hours to identify potential vulnerabilities.

Vocabulary - Ustálená slovní spojení času

📑 In the moment 🔱 In a moment

"In the moment" s určitým členem znamená, že se něco odehrává právě teď a tady bez ohledu na minulosti či budoucnost. Často se tak tato fráze používá pro momenty plného soustředění a meditačního pohroužení a popisuje tedy kvalitu v současnosti prožívaného okamžiku.

🧩 During meditation, it's important to stay in the moment to fully benefit from the practice.

🧩 He was completely in the moment when coding, oblivious to time passing by.

"In a moment" naproti tomu znamená, že se něco stane velmi brzy, téměř okamžitě - za malý moment. Jde o vyjádření blízké budoucnosti podobně jako "in a few minutes."

🧩 Hang on, I’ll be off the phone in a moment.

🧩 She said she would return in a moment with the documents we needed for the meeting.

📑 On time 🔱 In time

The phrases "on time" and "in time" are both used to refer to timing, but they have distinct meanings and are used in different contexts. "On time" means that something happens at the scheduled or expected time. It emphasizes punctuality and adherence to a specific schedule.

🧩 The train arrived on time, exactly at 10:00 AM.

🧩 We need to submit the project on time to meet the deadline.

🧩 The meeting started on time, as everyone was present.

"In time" means that something happens before a deadline or within a period that is soon enough to achieve a desired outcome. It often implies there is some flexibility, but the action still occurs before it is too late.

🧩 I got to the airport just in time for my flight.

🧩 She finished her assignment in time to review it before submitting.

🧩 He was able to fix the bug in time for the product launch.

📑 At the same time

Indicates that two or more events occur simultaneously.

🧩 The server crashed at the same time the software update was deployed.

📑 By the time

Used to specify a deadline or a point in time before which something will happen.

🧩 By the time the meeting starts, I will have completed the system analysis.

📑 In the meantime / meanwhile

Refers to the period between two events or the time during which something else is happening.

🧩 In the meantime, continue with your current tasks until the new guidelines are released.

🧩 Meanwhile, the development team will focus on fixing the reported bugs.

📑 From time to time

Means occasionally or not very often.

🧩 From time to time, we need to update our software to protect against new threats.

📑 At a set time

Specifies a specific, agreed-upon time when something will occur.

🧩 The network maintenance is scheduled at a set time to minimize disruption.

📑 For the time being

Indicates a temporary situation that is expected to change in the future.

🧩 For the time being, we will use the old database system until the new one is fully operational.

📑 Over time

Refers to changes that happen gradually throughout a period.

🧩 Over time, our software has evolved to meet changing customer needs."

📑 With time

Suggests that something will occur or become clear later, as time passes.

🧩 With time, you’ll become proficient in the new programming language.

📑 Ahead of time

Means to do something before the set time or earlier than scheduled.

🧩 We completed the network upgrades ahead of time, which was a great success.

📑 At all times

Indicates something that is continuous or always true.

🧩 The data center is monitored at all times for any unusual activity.

📑 From this time forward

Used to designate a starting point in time from which something will begin to apply or be true.

🧩 From this time forward, all intern access will be logged and monitored.

📑 X years / months / weeks / days from now

Happening X time frame in future from this point.

🧩 Five weeks from now, I hope to have completed my Python course.

🧩 Where do you see yourself living five years from now?

🧩 The project deadline is two weeks from now.

🧩 The new software update will be released ten weeks from now.

📑 Pass the time

To engage in an activity to make the time seem to go by more quickly, often while waiting for something.

🧩 She read a book to pass the time while waiting for her flight.

🧩 He started coding to pass the time during the lockdown.

🧩 Listening to podcasts helps me pass the time when I'm stuck in traffic.

Listening - Bash in 100 seconds

Bash, a command language interpreter for interacting with a computer from the command line. It's also called the Shell because it surrounds the operating system kernel to hide its intricate details while allowing you, the programmer, to do important stuff like access data and write files by typing simple commands.

This was a revolutionary concept when it was developed in the early 70s, back when programmers were still using punch cards. The shell concept evolved over the years with the Bourne Shell being the most popular version. That is until 1989 when the Bourne-Again Shell, or Bash, came about.

When you open up the terminal on a Unix machine like macOS and most Linux distros, the default shell is usually Bash. It provides a prompt where you can type a command, which will then be interpreted by the shell and executed on the operating system. To find out if you're running Bash, type in "which $SHELL" from the command line.

It's like any other application that lives in the binaries directory, but Bash is also a programming language that allows us to write scripts, which means anything we type manually into the command line can be automated with code. When you first launch the Shell, it actually runs a startup script that's defined in the bash profile or .bashrc file on your system. This allows you to customize the behavior and appearance of the Shell whenever you start a new session. You can add your own custom Bash scripts to any project by creating a file that ends in .sh or no file extension at all.

The first line in that file should always be a SHEBANG #! followed by the path to the application that should run it. Below that, we can start writing commands like "echo" to print something, and they'll be interpreted line-by-line.

To create a variable, type a name in all caps followed by the equal sign, then reference it later in the script using a dollar sign in front of the name. Now, to execute the script, simply type the file name into the Shell. That was easy, but what if we want to pass in some arguments when we run the script? Positional arguments will automatically be assigned variable names of $1, $2, $3 and so on. In other cases, you may need additional user input in the middle of a script. You can create loops in Bash like a do-while loop that will prompt the user to continue the script on a "yes" answer or exit on a "no" answer.

From there, we can implement conditional logic with an "if" statement, which will test if the value on the left side is less than the value on the right side. If true, then run this command; otherwise, run the "else" command. Another cool feature is that if you have multiple long-running processes, you can run them in parallel in the background by adding an ampersand after the command.
